## Quick Example

```bash
# Install
brew install chrismcdermut/proletariat/prlt

# Create workspace
prlt new

# Create a ticket
prlt ticket create --title "Add OAuth" --category feature

# Spawn an agent to work on it
prlt work spawn
```

Agent spawns in its own branch, writes code, opens PR. You review and merge.

## Key Features

<AccordionGroup>
  <Accordion title="Multi-Agent Orchestration">
    Work on multiple tickets simultaneously. Each agent operates in its own isolated workspace with dedicated git branches. Scale to 50+ concurrent agents depending on your machine.
  </Accordion>
  
  <Accordion title="Flexible Execution Modes">
    Run agents in Docker containers for security or on the host for speed. Choose terminal display for watching progress or background mode for detached operation.
  </Accordion>
  
  <Accordion title="Built-in MCP Server">
    Includes a Model Context Protocol server with 100+ tools. Integrate with Claude Code, Cursor, and other AI clients.
  </Accordion>
  
  <Accordion title="Themed Agent Naming">
    Customize agent names with built-in themes: billionaires (musk, bezos, gates), toyotas (camry, supra, tacoma), or companies (stripe, vercel, linear). Create your own custom themes.
  </Accordion>
  
  <Accordion title="Structured Workflow">
    Manage projects, epics, tickets, and specs. Tickets flow through configurable workflows with phases and statuses. Track all work in one SQLite database.
  </Accordion>
  
  <Accordion title="JSON Mode for AI Integration">
    Every command supports `--json` flag for machine-readable output. AI agents can parse prompts, make selections, and drive the CLI autonomously.
  </Accordion>
</AccordionGroup>

## Use Cases

### Parallel Feature Development

Spawn multiple agents to work on different features simultaneously:

```bash
prlt ticket create --title "Add OAuth" --category feature
prlt ticket create --title "Add API rate limiting" --category feature
prlt ticket create --title "Add email notifications" --category feature

# Spawn all three in parallel
prlt work spawn TKT-001 TKT-002 TKT-003 --mode docker

# Watch the board as they work
prlt board watch
```

### Bug Bash

Process multiple bugs at once:

```bash
# Spawn all bugs from backlog
prlt work spawn --all --column Backlog --category bug
```

### Code Review with Agents

Have an agent review and refine ticket requirements:

```bash
prlt work start TKT-042 --action groom
```
