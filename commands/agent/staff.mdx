Create and manage staff agents - permanent workspaces for long-term autonomous development.

## Subcommands

- [`prlt agent staff add`](#add) - Create new staff agents
- [`prlt agent staff list`](#list) - List all staff agents
- [`prlt agent staff remove`](#remove) - Remove staff agents

---

## add

Create new staff (persistent) agents in your workspace.

### Usage

```bash
prlt agent staff add [names...] [options]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `names` | Space-separated agent names (optional, prompts if not provided) |

### Options

| Option | Description |
|--------|-------------|
| `--theme`, `-t` | Pick names from specific theme (billionaires, toyotas, companies) |
| `--no-container` | Skip devcontainer setup (not recommended) |
| `--clone` | Use git clone instead of worktree (more isolation, no real-time sync) |
| `--machine` | Output in JSON format |

### Examples

#### Add agents from active theme

```bash
prlt agent staff add
```

Interactive selection from workspace's current theme:

```
? Select agents from Billionaires:
  ◯ altman
  ◉ bezos
  ◉ gates
  ◯ musk
  ◉ zuck
  ◯ Enter custom name(s)...

✓ Successfully added 3 agent(s): bezos, gates, zuck
  From theme: Billionaires
  Devcontainer config created for isolated execution
```

#### Add agents by name

```bash
prlt agent staff add altman musk
```

Creates agents directly:

```
✓ Successfully added 2 agent(s): altman, musk
  Devcontainer config created for isolated execution
```

#### Add from specific theme

```bash
prlt agent staff add --theme toyotas
```

Select from Toyota vehicle names:

```
? Select agent names from Toyota Garage:
  ◉ camry
  ◯ corolla
  ◉ tacoma
  ◯ tundra
  ◉ 4runner

✓ Successfully added 3 agent(s): camry, tacoma, 4runner
  From theme: Toyota Garage
  Devcontainer config created for isolated execution
```

#### Add without devcontainer

```bash
prlt agent staff add agent-1 --no-container
```

Creates agent workspace without Docker isolation:

```
✓ Successfully added 1 agent(s): agent-1
```

<Warning>
  Using `--no-container` means agents run directly on your machine.
  Recommended only for testing or non-autonomous workflows.
</Warning>

#### Add with git clone

```bash
prlt agent staff add research-agent --clone
```

Uses independent git clone instead of worktree:

- ✅ Complete isolation from main repository
- ✅ Can check out different commits/branches
- ❌ No real-time sync with HQ
- ❌ Uses more disk space

### Custom Names

When prompted, select "Enter custom name(s)..." to provide your own:

```
? Enter custom agent names (space-separated):
> dev-1 research testing

Normalized names:
   dev-1 → dev-1 ✓
   research → research ✓
   testing → testing ✓

✓ Successfully added 3 agent(s): dev-1, research, testing
```

Agent names must be:
- Lowercase alphanumeric
- Optional hyphens or underscores
- Start with letter or number

### Name Validation

Invalid names are normalized or rejected:

```bash
prlt agent staff add "Agent 1" AGENT-2 agent_3

Normalized names:
   Agent 1 → agent-1
   AGENT-2 → agent-2

✓ Successfully added 3 agent(s): agent-1, agent-2, agent_3
```

### Staff vs Temp Agents

| Feature | Staff (Persistent) | Temp (Ephemeral) |
|---------|-------------------|------------------|
| **Creation** | Manual via `prlt agent staff add` | Auto via `prlt work spawn` |
| **Naming** | From themes or custom | Auto-generated (bold-bezos) |
| **Lifetime** | Permanent until removed | Cleaned after work completes |
| **Location** | `staff/` directory | `temp/` directory |
| **Use case** | Long-term development | Single task execution |
| **History** | Tracks completed tickets | No history |

## remove

Remove staff agents from the workspace.

<Warning>
  This command removes the agent workspace directory and all its contents.
  Any uncommitted work will be lost. Ensure work is committed and pushed before removing.
</Warning>

### Usage

```bash
prlt agent staff remove [names...]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `names` | Space-separated agent names to remove (prompts if not provided) |

### Examples

```bash
# Remove specific agents
prlt agent staff remove gates zuck

# Interactive selection
prlt agent staff remove
```
