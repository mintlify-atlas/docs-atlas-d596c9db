## Core Commands

These commands are available at the root level:

- [`prlt init`](/commands/workspace/init) - Initialize machine-level configuration
- [`prlt new`](/commands/workspace/new) - Create a new HQ workspace
- `prlt whoami` - Display current workspace and agent info
- `prlt ps` - Show running executions (alias for `execution list`)

## JSON Mode

Most commands support `--json` mode for AI agent orchestration. When enabled:

- Interactive prompts return JSON prompt configs instead of blocking
- Output is structured and parseable
- AI agents can navigate command flows programmatically

```bash
prlt ticket create --json
```

Returns:
```json
{
  "prompt": {
    "type": "input",
    "name": "title",
    "message": "Enter ticket title:"
  },
  "metadata": {
    "command": "ticket create",
    "flags": {}
  }
}
```

See the [JSON Mode guide](/guides/json-mode) for details.

## Exit Codes

All commands follow standard Unix exit code conventions:

- `0` - Success
- `1` - General error
- `2` - Invalid usage (missing arguments, invalid flags)

## Getting Help

For command-specific help:

```bash
prlt <command> --help
```

For interactive command menus:

```bash
prlt <namespace>
```

Example:
```bash
prlt agent
# Shows interactive menu with all agent commands
```
