---
title: prlt execution stop
description: Stop running agent work executions
---

Stop active agent executions gracefully or forcefully, either individually or in bulk.

## Basic Usage

### Stop Execution

Stop specific execution:

```bash
prlt execution stop WORK-023
```

Output:

```
Stopping WORK-023...

✓ Stopped execution WORK-023
   Ticket: TKT-042
   Agent: alice
```

### Interactive Selection

Choose execution from running list:

```bash
prlt execution stop
```

Prompts:
```
Select execution to stop:
  WORK-023 - TKT-042 (alice, devcontainer)
  WORK-021 - TKT-040 (charlie, devcontainer)
```

## Flags

<ParamField path="--force" type="boolean" default="false">
  Force kill (SIGKILL instead of SIGTERM)
</ParamField>

<ParamField path="-f" type="boolean">
  Shorthand for --force
</ParamField>

<ParamField path="--all" type="boolean" default="false">
  Stop all running executions
</ParamField>

<ParamField path="--agent" type="string">
  Stop all executions for specific agent
</ParamField>

<ParamField path="-a" type="string">
  Shorthand for --agent
</ParamField>

<ParamField path="--json" type="boolean" default="false">
  Output structured JSON (for interactive selection)
</ParamField>

## Arguments

<ParamField path="id" type="string">
  Execution ID (WORK-XXX). Prompts if not provided (ignored if --all or --agent used).
</ParamField>

## Examples

### Stop Single Execution

Graceful shutdown:

```bash
prlt execution stop WORK-023
```

Sends SIGTERM, allows cleanup:
- Agent saves state
- Logs flush
- Container stops gracefully

### Force Stop

Immediate termination:

```bash
prlt execution stop WORK-023 --force
```

Sends SIGKILL:
- No cleanup
- Immediate termination
- Use when graceful stop fails

### Stop All

Stop all running executions:

```bash
prlt execution stop --all
```

Output:

```
Stopping 3 execution(s)...

   ✓ WORK-023 (alice → TKT-042)
   ✓ WORK-021 (charlie → TKT-040)
   ✓ WORK-019 (bob → TKT-038)

Summary
   Stopped: 3
```

### Stop by Agent

Stop all work for specific agent:

```bash
prlt execution stop --agent alice
```

Output:

```
Stopping 2 execution(s)...

   ✓ WORK-023 (alice → TKT-042)
   ✓ WORK-020 (alice → TKT-039)

Summary
   Stopped: 2
```

### Force Stop All

Force kill all running work:

```bash
prlt execution stop --all --force
```

Use when agents are unresponsive.

## Stop Behavior

### Graceful Stop (default)

Sends SIGTERM signal:

1. **Kill tmux session**: All environments
2. **Kill process**: Host executions with PID
3. **Update status**: Mark as stopped in database
4. **Container stays**: Devcontainers remain for reuse

**Example:**

```bash
prlt execution stop WORK-023
```

### Force Stop

Sends SIGKILL signal:

1. **Kill session**: Immediate termination
2. **Kill process**: SIGKILL if PID available
3. **Update status**: Mark as stopped
4. **No cleanup**: Agent doesn't save state

**Example:**

```bash
prlt execution stop WORK-023 --force
```

Use when:
- Graceful stop doesn't work
- Agent is hung
- Emergency shutdown needed

## Environment Handling

### Host Execution

Stops:
- Tmux session: `tmux kill-session -t prlt-alice-work-023`
- Process: `kill <pid>` (if PID available)

### Devcontainer Execution

Stops:
- Tmux inside container: `docker exec <id> tmux kill-session`
- Container stays running (can be reused)

### Docker Execution

Stops:
- Tmux session inside container
- Container stays running

### VM Execution

Manual stop required:

```
Cannot automatically stop VM execution.
SSH to vm-host and stop manually.
```

## Bulk Operations

### Stop All Running

```bash
prlt execution stop --all
```

Stops all executions with status:
- `starting`
- `running`

Skips:
- `completed`
- `failed`
- `stopped`

### Stop by Agent

```bash
prlt execution stop --agent alice
```

Stops all alice's running work.

### Force Stop All

```bash
prlt execution stop --all --force
```

Force kills everything.

## Status Updates

After stopping, execution status changes to `stopped`:

**Before:**
```
WORK-023  TKT-042  alice  devcontainer  running
```

**After:**
```
WORK-023  TKT-042  alice  devcontainer  stopped
```

Verify:
```bash
prlt execution list --status stopped
```

## Error Handling

### Not Running

Already stopped execution:

```bash
prlt execution stop WORK-020
```

Output:
```
Execution WORK-020 is not running (status: completed).
```

### Not Found

Invalid execution ID:

```bash
prlt execution stop WORK-999
```

Output:
```
Execution "WORK-999" not found.
```

### No Running Executions

Nothing to stop:

```bash
prlt execution stop --all
```

Output:
```
No running executions found.
```

### Partial Failures

Some stops fail:

```
Stopping 3 execution(s)...

   ✓ WORK-023 (alice → TKT-042)
   ✗ WORK-021: Container not found
   ✓ WORK-019 (bob → TKT-038)

Summary
   Stopped: 2
   Failed: 1
```

## Use Cases

### Emergency Stop

Agent going wrong direction:

```bash
prlt execution stop WORK-023
```

Stops work immediately. Ticket remains in current state.

### Agent Cleanup

Stop all work before shutdown:

```bash
prlt execution stop --all
```

### Resource Reclaim

Free up agent for other work:

```bash
prlt execution stop --agent alice
```

Stops alice's work so she can start new ticket.

### Hung Execution

Agent stopped responding:

```bash
prlt execution stop WORK-023 --force
```

Force kill to clean up.

### Development Session End

Stop all work when done:

```bash
prlt execution stop --all
prlt docker clean
```

Clean slate for next session.

## After Stopping

### Check Status

Verify execution stopped:

```bash
prlt execution list --status stopped
```

### Review Logs

See what agent did:

```bash
prlt execution logs WORK-023 --tail 100
```

### Clean Up Containers

Remove stopped containers:

```bash
prlt docker clean
```

### Restart Work

Spawn agent on same ticket:

```bash
prlt spawn alice TKT-042
```

Agent continues from ticket's current state.

## Docker Interaction

### Containers Stay Running

Stopping execution doesn't stop devcontainer:

```bash
prlt execution stop WORK-023
docker ps  # Container still running
```

Containers can be reused for future work.

### Manual Container Stop

Stop container separately:

```bash
prlt docker stop abc123
```

### Clean All

Stop executions and remove containers:

```bash
prlt execution stop --all
prlt docker clean
```

## Tips

<Tip>
Use graceful stop (default) first. Only use --force if agent doesn't respond within 10 seconds.
</Tip>

<Tip>
Stop all work before shutting down to avoid orphaned processes: `prlt execution stop --all`
</Tip>

<Tip>
After stopping, check logs to see progress before interruption: `prlt execution logs WORK-023 --tail 50`
</Tip>

<Warning>
Stopping an execution doesn't revert changes. Agent's work remains in the codebase. Use git to undo if needed.
</Warning>
