title: prlt work ready
description: Mark work as ready for review and optionally create a pull request
---

## Usage

```bash
prlt work ready [TICKET-ID] [OPTIONS]
```

Marks work as ready for review by moving the ticket to the Review column. Optionally creates a pull request if the work is on a feature branch. Automatically terminates running executions and updates the board.

## Arguments

<ParamField path="TICKET-ID" type="string" optional>
  Ticket ID to mark as ready. If omitted, displays an interactive dropdown to select from in-progress tickets.
</ParamField>

## Pull Request Options

<ParamField path="--pr" type="boolean" default="false">
  Create a pull request for this work. Requires `gh` CLI to be installed and authenticated.
</ParamField>

<ParamField path="--draft-pr" type="boolean" default="false">
  Create a draft pull request (implies `--pr`). Useful for work that's ready for early feedback but not final review.
</ParamField>

<ParamField path="--no-pr" type="boolean" default="false">
  Skip PR creation prompt. Use this to explicitly decline PR creation when running interactively.
</ParamField>

## Other Options

<ParamField path="--project" type="string">
  Filter tickets by project ID (only shows tickets from specified project in dropdown).
</ParamField>

<ParamField path="--json" type="boolean" default="false">
  Output result as JSON for programmatic use.
</ParamField>

## Behavior

1. **Status Transition**: Moves ticket from current status to Review column
2. **Execution Cleanup**: Marks any running executions as completed
3. **PR Creation** (if requested):
   - Detects current branch from execution record or git
   - Pushes unpushed commits to origin
   - Generates PR title from ticket ID and title
   - Generates PR body from ticket description and commit log
   - Creates PR using `gh` CLI
   - Stores PR URL in ticket metadata
4. **Board Sync**: Auto-exports to `board.md` if configured

## Column Configuration

The "Review" column is configurable via workspace settings:

```bash
# Set custom Review column name
prlt config set column_review "In Review"

# View current setting
prlt config get column_review
```

Default: `"Review"`

## Pull Request Requirements

For PR creation to work:

1. **GitHub CLI**: Install with `brew install gh` (macOS) or [other platforms](https://cli.github.com/)
2. **Authentication**: Run `gh auth login` to authenticate
3. **Feature Branch**: Must be on a branch other than main/master
4. **Commits**: Must have commits to push (work to review)
5. **Remote**: Branch must be pushed to origin

## Examples

### Interactive Mode

```bash
# Prompt to select in-progress ticket
prlt work ready

# Interactive with PR creation prompt
prlt work ready  # Will prompt "Create a pull request for this work?"
```

### Direct Commands

```bash
# Mark specific ticket as ready (no PR)
prlt work ready TKT-001

# Mark ready and create PR
prlt work ready TKT-001 --pr

# Create draft PR
prlt work ready TKT-001 --draft-pr

# Explicitly skip PR
prlt work ready TKT-001 --no-pr
```

### With Project Filter

```bash
# Ready ticket in specific project
prlt work ready TKT-001 --project PROJ-123

# Interactive with project filter
prlt work ready --project PROJ-123
```

### JSON Output

```bash
# Get result as JSON
prlt work ready TKT-001 --pr --json
```

   To: Review

```

```

## Selection Criteria

When running without a ticket ID, the command filters tickets to show only those eligible for review:

- Tickets with status category `started`
- Tickets with status name containing "progress" (case-insensitive)
- Optionally filtered by `--project` flag

If no eligible tickets are found, displays: `"No in-progress work found."`

## Pull Request Details

### PR Title Format

```
[TICKET-ID] Ticket Title
```

Example: `[TKT-001] Implement user authentication`

### PR Body Format

The PR body includes:

1. **Ticket Link**: Link to ticket (if external issue integration)
2. **Ticket Description**: Full ticket description
3. **Commit Log**: Up to 10 most recent commits
4. **Metadata**: Ticket ID, category, priority

### PR Behavior

- **Existing PR**: If PR already exists for the branch:
  - Merged PRs: Show existing PR URL, skip creation
  - Open PRs: Push unpushed commits to update PR, show existing PR URL
- **Unpushed Commits**: Automatically pushes before creating PR
- **Base Branch**: Automatically detects base branch (main or master)

## Execution Termination

When marking work ready:

1. Searches for running executions associated with the ticket
2. Updates execution status from `running` to `completed`
3. Records completion timestamp
4. Uses execution record to find branch name for PR creation

## Error Handling

<ResponseField name="NOT_IN_WORKSPACE">
  Command must be run from within a workspace. Run `prlt new` to create one.
</ResponseField>

<ResponseField name="TICKET_NOT_FOUND">
  Specified ticket ID does not exist. Check ticket ID or run without arguments to select interactively.
</ResponseField>

<ResponseField name="COLUMN_NOT_FOUND">
  Review column not found in board configuration. Configure with `prlt config set column_review <column-name>`.
</ResponseField>

<ResponseField name="NO_IN_PROGRESS_WORK">
  No in-progress work found (in JSON mode).
</ResponseField>

### PR Creation Warnings

- `"Could not determine current branch. Skipping PR creation."` - Not on a git branch
- `"Failed to push branch. Skipping PR creation."` - Git push failed
- `"Failed to push commits. Skipping PR creation."` - Git push failed for unpushed commits
- `"Failed to create PR: <error>"` - GitHub API error
