Display comprehensive details for a specific ticket.

## Usage

<CodeGroup>

```bash With Ticket ID
prlt ticket view TKT-001
```

```bash Interactive Selection
prlt ticket view
```

```bash JSON Output
prlt ticket view TKT-001 --json
```

</CodeGroup>

## Arguments

<ParamField path="ticketId" type="string">
  Ticket ID to view. If not provided, shows an interactive dropdown to select a ticket.
  
  ```bash
  prlt ticket view TKT-001
  ```
</ParamField>

## Flags

<ParamField path="--project" type="string">
  Project ID to filter tickets (when using interactive mode).
  
  ```bash
  prlt ticket view --project mobile-app
  ```
</ParamField>

<ParamField path="--json" type="boolean">
  Output ticket details as JSON.
  
  ```bash
  prlt ticket view TKT-001 --json
  ```
</ParamField>

## Interactive Mode

When run without a ticket ID, displays a searchable dropdown of all tickets:

```
? Select ticket to view:
  TKT-001 - Fix login bug (Backlog)
  TKT-002 - Add dark mode (In Progress)
  TKT-003 - Update docs (Backlog)
  TKT-004 - Refactor auth (In Progress)
```

Use arrow keys to navigate and press Enter to select.

## Displayed Fields

The command shows all available ticket information:

### Core Fields

- **ID**: Unique ticket identifier
- **Title**: Short summary
- **Project**: Project name (or ID if project was deleted)
- **Status**: Current column/status name
- **Priority**: Priority level (P0-P3 or workspace-defined)
- **Category**: Ticket category (bug, feature, etc.)

### Timestamps

- **Created**: When the ticket was created
- **Updated**: Last modification time

### Optional Fields

These fields are shown when present:

- **Description**: Full ticket description with formatting
- **Owner**: Human responsible for the ticket
- **Assignee**: Who's executing (human or agent)
- **Epic**: Parent epic ID
- **Branch**: Git branch for this ticket's work
- **Labels**: Tags applied to the ticket
- **Subtasks**: Checklist items with completion status
- **Acceptance Criteria**: Structured verifiable criteria
- **Blocked By**: Other ticket IDs this depends on
- **Spec**: Linked specification ID
- **Metadata**: Custom key-value pairs

## Understanding Ticket Structure

<AccordionGroup>

<Accordion title="Status vs Status Category">
  - **statusName**: The display name of the column (e.g., "In Progress", "Ready for Review")
  - **statusCategory**: The semantic state (e.g., "started", "completed", "backlog")
  
  Status categories are fixed: `triage`, `backlog`, `unstarted`, `started`, `completed`, `canceled`
</Accordion>

<Accordion title="Owner vs Assignee">
  - **owner**: The human responsible for the ticket (product owner, PM, etc.)
  - **assignee**: Who's actively working on it (human developer or AI agent)
  
  A ticket can have an owner without an assignee (not yet assigned) or vice versa.
</Accordion>

<Accordion title="Subtasks vs Acceptance Criteria">
  - **subtasks**: Simple checklist items (id, title, done)
  - **acceptanceCriteria**: Structured criteria parsed from description
  
  Both are tracked independently and can be marked as complete.
</Accordion>

<Accordion title="Epic vs Spec">
  - **epicId**: Groups related tickets for tracking (e.g., "OAuth Implementation")
  - **specId**: Links to formal specification document
  
  A ticket can be in an epic, linked to a spec, both, or neither.
</Accordion>

</AccordionGroup>

## Use Cases

### Before Starting Work

Review ticket details before spawning an agent:

```bash
prlt ticket view TKT-001
# Review description and acceptance criteria
prlt work spawn TKT-001
```

### During Code Review

Check the original requirements:

```bash
prlt ticket view TKT-001 --json | jq -r '.ticket.description'
```

### For Reporting

Extract ticket data for reports:

```bash
# Get all P0 tickets with full details
for ticket in $(prlt ticket list --priority P0 --json | jq -r '.[].id'); do
  prlt ticket view $ticket --json >> p0-tickets.json
done
```

### Debugging Agent Issues

Verify what the agent was supposed to do:

```bash
prlt ticket view TKT-001
prlt execution logs <execution-id>
```
