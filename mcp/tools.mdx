## Overview

The Proletariat MCP server provides **140+ tools** organized into logical categories. Each tool accepts structured parameters and returns JSON responses.

<Info>
  All tools use Zod schemas for validation. Optional parameters are clearly marked, and error responses follow a consistent format.
</Info>

## Tool Categories

### Ticket Management (15 tools)

Core ticket operations for creating, querying, and managing work items.

<AccordionGroup>
  <Accordion title="ticket_list" icon="list">
    List tickets with optional filters. Returns summary fields only.
    
    **Parameters:**
    - `project` (optional): Project ID
    - `column` (optional): Filter by status/column
    - `priority` (optional): Filter by priority
    - `category` (optional): Filter by category
    - `assignee` (optional): Filter by assignee
    - `owner` (optional): Filter by owner
    - `search` (optional): Search in title/description
    - `epic` (optional): Filter by epic ID
    - `label` (optional): Filter by label name
    - `label_group` (optional): Filter by label group
    - `all_projects` (optional): List from all projects
    - `limit` (optional): Max results (default: 50)
    - `offset` (optional): Skip N tickets (default: 0)
    
    **Returns:** Array of ticket summaries with id, title, priority, category, status, labels.
  </Accordion>
  
  <Accordion title="ticket_show" icon="eye">
    Get full details for a specific ticket, including description, acceptance criteria, and subtasks.
    
    **Parameters:**
    - `id` (required): Ticket ID (e.g., "TKT-042")
    
    **Returns:** Complete ticket object with all fields.
  </Accordion>
  
  <Accordion title="ticket_create" icon="plus">
    Create a new ticket.
    
    **Parameters:**
    - `title` (required): Ticket title
    - `project` (optional): Project ID
    - `description` (optional): Full description
    - `priority` (optional): Priority level
    - `category` (optional): Category (feature, bug, etc.)
    - `column` (optional): Initial status/column
    - `assignee` (optional): Assigned user
    - `owner` (optional): Owner
    - `epic_id` (optional): Link to epic
    - `labels` (optional): Array of label names
    - `subtasks` (optional): Array of subtask titles
    
    **Returns:** Created ticket with ID.
  </Accordion>
  
  <Accordion title="ticket_update" icon="pen">
    Update an existing ticket's fields.
    
    **Parameters:**
    - `id` (required): Ticket ID
    - `title` (optional): New title
    - `description` (optional): New description
    - `priority` (optional): New priority
    - `category` (optional): New category
    - `assignee` (optional): New assignee
    - `owner` (optional): New owner
    - `acceptance_criteria` (optional): Array of criteria
    - `subtasks` (optional): Array of subtasks
    
    **Returns:** Updated ticket.
  </Accordion>

### Work & Agent Operations (12 tools)

Spawn agents, track executions, and manage the work lifecycle.

<AccordionGroup>

## Response Formats

All tools return responses in this structure:

### Success Response

```json
{
  "content": [
    {
      "type": "text",
      "text": "{\"success\": true, \"data\": {...}}"
    }
  ]
}
```

### Error Response

```json
{
  "content": [
    {
      "type": "text",
      "text": "{\"success\": false, \"error\": \"Error message\"}"
    }
  ],
  "isError": true
}
```

## Usage Examples

### Query Tickets and Spawn Agent

```typescript
// Find high-priority bugs
const result = await client.callTool('ticket_list', {
  category: 'bug',
  priority: 'P1',
  column: 'Backlog'
})

const tickets = JSON.parse(result.content[0].text)

// Spawn an agent on the first one
if (tickets.tickets.length > 0) {
  await client.callTool('work_start', {
    ticket_id: tickets.tickets[0].id,
    action: 'implement',
    mode: 'docker',
    display: 'background'
  })
}
```

### Monitor Running Work

```typescript
// Check all running executions
const executions = await client.callTool('execution_list', {
  status: 'running'
})

// Get logs for one
const logs = await client.callTool('execution_logs', {
  id: 'exec-123',
  lines: 100
})
```

### Batch Operations

```typescript
// Get all tickets needing grooming
const tickets = await client.callTool('ticket_list', {
  column: 'Backlog',
  limit: 10
})

// Spawn groom actions on all of them
const ticketIds = JSON.parse(tickets.content[0].text).tickets.map(t => t.id)

await client.callTool('work_spawn_batch', {
  ticket_ids: ticketIds,
  action: 'groom',
  mode: 'docker'
})
```

## Tool Naming Convention

Tools follow this naming pattern:

```
<resource>_<operation>
```

Examples:
- `ticket_list`, `ticket_create`, `ticket_show`
- `work_start`, `work_complete`, `work_ready`
- `agent_list`, `agent_status`, `agent_add`

This makes tools predictable and easy to discover.
