This guide walks you through creating your first workspace, ticket, and spawning an AI agent to do work.

## Overview

Proletariat orchestrates AI agents to work on your codebase. Here's the basic workflow:

1. **Create an HQ** - Your workspace that manages repos, agents, and tickets
2. **Create tickets** - Define work with structured requirements
3. **Spawn agents** - Launch AI workers that create branches, write code, and open PRs

<Note>
  Make sure you've completed the [installation](/installation) and authenticated with `claude login` before continuing.
</Note>

## Create Your First HQ

An HQ (headquarters) is your workspace for managing repositories, agents, and projects.

<Steps>
  <Step title="Run the setup command">
    ```bash
    prlt new
    ```

    This starts an interactive setup wizard.
  </Step>

  <Step title="Configure your HQ">
    The wizard will prompt you for:

    - **HQ name**: Choose a name for your workspace (e.g., `my-project`)
    - **Location**: Where to create the HQ (defaults to `./{name}-hq`)
    - **Agent theme**: Choose how agents are named:
      - `billionaires` - Tech founders like `musk`, `gates`, `bezos` (default)
      - `toyotas` - Toyota models like `camry`, `supra`, `tacoma`
      - `companies` - Tech companies like `stripe`, `vercel`, `linear`
    - **Initial agents**: Add staff agents (optional)
    - **Repositories**: Add existing repos or clone new ones
    - **PMO setup**: Initialize project management structure
  </Step>

  <Step title="Complete setup">
    Proletariat creates your HQ structure:

    ```
    my-project-hq/
    ├── .proletariat/
    │   └── workspace.db        # State database
    ├── repos/                  # Your repositories
    └── agents/                 # Agent workspaces
        ├── staff/              # Named agents
        └── temp/               # Ephemeral agents
    ```
  </Step>
</Steps>

<Tip>
  The billionaires theme is popular - finally, they work for us!
</Tip>

## Create Your First Ticket

Tickets provide structured context for agents. They include requirements, acceptance criteria, and priority.

<Steps>
  <Step title="Create a ticket">
    ```bash
    prlt ticket create --title "Add OAuth authentication" --category feature
    ```

    <Note>
      You can also run `prlt ticket create` without flags for an interactive prompt.
    </Note>
  </Step>

  <Step title="Add details (optional)">
    Create a more detailed ticket with priority and description:

    ```bash
    prlt ticket create \
      --title "Add OAuth authentication" \
      --description "Implement Google and GitHub OAuth login" \
      --priority P1 \
      --category feature
    ```
  </Step>

  <Step title="View your ticket">
    ```bash
    prlt ticket view TKT-001
    ```

    Or see all tickets:

    ```bash
    prlt ticket list
    ```
  </Step>
</Steps>

### Ticket Categories

Common categories include:
- `feature` - New functionality
- `bug` - Bug fixes
- `refactor` - Code improvements
- `test` - Testing work
- `docs` - Documentation

## Spawn Your First Agent

Now comes the magic - spawn an AI agent to work on your ticket.

<Steps>
  <Step title="Start the spawn wizard">
    ```bash
    prlt work spawn
    ```

    This launches an interactive menu.
  </Step>

  <Step title="Select tickets">
    Choose one or more tickets to work on. Tickets are grouped by priority (P0, P1, P2, etc.).
  </Step>

  <Step title="Choose environment">
    Select where the agent runs:

    - **Docker** - Fully isolated in a container (recommended for safety)
    - **Host** - Runs directly on your machine (faster, less isolation)
  </Step>

  <Step title="Select action">
    Choose what the agent should do:

    - `implement` - Write code to complete the ticket
    - `groom` - Refine requirements and add acceptance criteria
    - `review` - Review existing code for issues
    - `custom` - Provide your own prompt
  </Step>

  <Step title="Agent starts working">
    The agent:
    1. Creates a new branch (e.g., `feat/TKT-001-oauth`)
    2. Sets up an isolated workspace
    3. Reads the ticket requirements
    4. Writes code and commits changes
    5. Opens a pull request
  </Step>
</Steps>

### Spawn Specific Tickets

Skip the interactive menu by specifying tickets directly:

```bash
prlt work spawn TKT-001 --action implement --mode docker
```

### Spawn Multiple Tickets

Work on multiple tickets in parallel:

```bash
prlt work spawn TKT-001 TKT-002 TKT-003
```

Each agent works in its own isolated branch - no conflicts!

<Warning>
  Parallel execution depends on your machine resources. 50+ concurrent agents is achievable with sufficient CPU and RAM.
</Warning>

## Monitor Agent Progress

While agents work, you can monitor their progress.

### View Running Agents

```bash
prlt execution list
```

Shows all active agents with their tickets, status, and runtime.

### Check the Board

```bash
prlt board
```

Displays a Kanban-style board with tickets organized by column (Backlog, In Progress, Review, Done).

### Watch in Real-Time

```bash
prlt board watch
```

Live updates as agents move tickets through workflow.

### View Agent Logs

```bash
prlt execution logs <agent-name>
```

See detailed output from a specific agent.

## Execution Modes

### Environment

Control where agents run:

<CodeGroup>
  ```bash Docker (default)
  prlt work spawn TKT-001
  # Runs in isolated Docker container
  ```

  ```bash Host
  prlt work spawn TKT-001 --run-on-host
  # Runs directly on your machine (faster)
  ```
</CodeGroup>

### Display Mode

Control how you see agent work:

<CodeGroup>
  ```bash Terminal (new tab)
  prlt work spawn TKT-001 --display terminal
  # Opens new terminal tab to watch agent
  ```

  ```bash Background (detached)
  prlt work spawn TKT-001 --display background
  # Agent works in background, reattach later
  ```
</CodeGroup>

### Permission Mode

Control agent autonomy:

<CodeGroup>
  ```bash Safe (default)
  prlt work spawn TKT-001
  # Agent asks permission for operations
  ```

  ```bash YOLO (autonomous)
  prlt work spawn TKT-001 --skip-permissions
  # Agent runs with full autonomy (safe in Docker)
  ```
</CodeGroup>

<Tip>
  Combine Docker + YOLO for fully autonomous agents that are safely sandboxed.
</Tip>

## Review and Merge

When an agent completes work:

<Steps>
  <Step title="Agent opens PR">
    The agent creates a pull request on GitHub with:
    - All code changes
    - Commit messages
    - PR description referencing the ticket
  </Step>

  <Step title="Review the changes">
    Review the PR on GitHub as you would any other PR:
    - Check code quality
    - Verify requirements are met
    - Request changes if needed
  </Step>

  <Step title="Merge or revise">
    - **Merge** if the work looks good
    - **Request changes** and the agent can revise:
      ```bash
      prlt work revise TKT-001 --feedback "Add error handling"
      ```
  </Step>
</Steps>

## Common Workflows

### Parallel Feature Development

Work on multiple features simultaneously:

```bash
# Create tickets
prlt ticket create --title "Add OAuth" --category feature
prlt ticket create --title "Add rate limiting" --category feature
prlt ticket create --title "Add notifications" --category feature

# Spawn all three in parallel
prlt work spawn TKT-001 TKT-002 TKT-003 --mode docker

# Watch progress
prlt board watch
```

### Bug Bash

Fix multiple bugs at once:

```bash
# Spawn all bugs in the Backlog
prlt work spawn --all --column Backlog --category bug
```

### Grooming Session

Have an agent refine ticket requirements:

```bash
prlt work spawn TKT-042 --action groom
```

The agent adds acceptance criteria, subtasks, and estimates.

## JSON Mode for AI Agents

Proletariat has built-in support for AI agent orchestration via `--json` mode:

```bash
prlt work spawn --json
```

Returns:
```json
{
  "prompt": {
    "type": "list",
    "message": "Select ticket to work on:",
    "choices": [
      {
        "name": "[P1] TKT-001 - Add OAuth authentication",
        "value": "TKT-001",
        "command": "prlt work start TKT-001 --json"
      }
    ]
  }
}
```

AI agents can parse this output, make selections, and call subsequent commands.
