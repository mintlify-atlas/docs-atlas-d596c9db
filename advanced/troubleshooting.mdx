This guide covers common issues and their solutions when using Proletariat.

## Installation Issues

<Accordion title="Command Not Found: prlt">

**Symptom:** `command not found: prlt` after installation

**Solutions:**

1. **Verify installation:**
   ```bash
   npm list -g @proletariat/cli
   ```

2. **Check npm global path:**
   ```bash
   npm config get prefix
   # Add to PATH if needed:
   export PATH="$(npm config get prefix)/bin:$PATH"
   ```

3. **Reinstall:**
   ```bash
   npm uninstall -g @proletariat/cli
   npm install -g @proletariat/cli
   ```

4. **Add to shell profile:**
   ```bash
   # ~/.zshrc or ~/.bashrc
   export PATH="$(npm config get prefix)/bin:$PATH"
   ```

</Accordion>

<Accordion title="Native Module Build Errors">

**Symptom:** `better_sqlite3.node: not a mach-o file` or similar

**Cause:** Compiled native module incompatible with Node.js version

**Solution:**

```bash
# Check active runtime
node -p "process.version + ' abi=' + process.versions.modules + ' ' + process.platform + '-' + process.arch"

# Rebuild native modules
npm rebuild better-sqlite3

# Validate
node -e "const Database=require('better-sqlite3'); const db=new Database(':memory:'); db.close(); console.log('better-sqlite3 OK')"
```

If validation fails, reinstall:

```bash
npm uninstall -g @proletariat/cli
npm install -g @proletariat/cli
```

**Supported Node versions:** 20, 22, 23, 24, 25

**Supported platforms:**
- `darwin-arm64` (Apple Silicon)
- `darwin-x64` (Intel Mac)
- `linux-arm64` (ARM Linux)
- `linux-x64` (x86_64 Linux)

</Accordion>

<Accordion title="Permission Denied During Install">

**Symptom:** `EACCES: permission denied`

**Solutions:**

1. **Use npm prefix (recommended):**
   ```bash
   mkdir ~/.npm-global
   npm config set prefix '~/.npm-global'
   export PATH=~/.npm-global/bin:$PATH
   npm install -g @proletariat/cli
   ```

2. **Fix npm permissions:**
   ```bash
   sudo chown -R $(whoami) $(npm config get prefix)/{lib/node_modules,bin,share}
   ```

<Warning>
Avoid using `sudo npm install -g` - use npm prefix instead.
</Warning>

</Accordion>

## Workspace Issues

<Accordion title="Database Not Found">

**Symptom:** `Database not found at ... Run 'prlt new' first`

**Cause:** Not in a prlt workspace or PRLT_HQ_PATH incorrect

**Solutions:**

1. **Initialize workspace:**
   ```bash
   prlt new
   ```

2. **Check current directory:**
   ```bash
   pwd
   ls -la .proletariat/
   ```

3. **Set HQ path explicitly:**
   ```bash
   export PRLT_HQ_PATH=/path/to/your/hq
   ```

4. **Add to shell profile:**
   ```bash
   # ~/.zshrc or ~/.bashrc
   export PRLT_HQ_PATH=/path/to/your/hq
   ```

</Accordion>

<Accordion title="Multiple Workspace Databases">

**Symptom:** Different tickets/data in different directories

**Cause:** Multiple `.proletariat/` directories created

**Solution:**

1. Find all workspace databases:
   ```bash
   find ~ -name "workspace.db" 2>/dev/null
   ```

2. Set explicit HQ path:
   ```bash
   export PRLT_HQ_PATH=/path/to/correct/hq
   ```

3. Remove duplicate databases:
   ```bash
   # Backup first!
   mv /path/to/duplicate/.proletariat /path/to/duplicate/.proletariat.backup
   ```

</Accordion>

<Accordion title="Corrupted Database">

**Symptom:** `SqliteError` or unexpected query results

**Solutions:**

1. **Check database integrity:**
   ```bash
   sqlite3 .proletariat/workspace.db "PRAGMA integrity_check;"
   ```

2. **Backup and recreate (last resort):**
   ```bash
   # BACKUP FIRST!
   cp .proletariat/workspace.db .proletariat/workspace.db.backup
   
   # Export data
   prlt ticket list --json > tickets-backup.json
   
   # Recreate
   mv .proletariat/workspace.db .proletariat/workspace.db.old
   prlt new
   ```

<Warning>
Database corruption is rare. Contact support before recreating.
</Warning>

</Accordion>

## Docker Issues

<Accordion title="Docker Daemon Not Running">

**Symptom:** `Cannot connect to the Docker daemon`

**Solutions:**

**macOS:**
```bash
open -a Docker
# Wait for Docker Desktop to start
```

**Linux:**
```bash
sudo systemctl start docker
sudo systemctl enable docker  # Auto-start on boot
```

**Verify:**
```bash
docker info
prlt docker status
```

</Accordion>

<Accordion title="Permission Denied (Docker)">

**Symptom:** `permission denied while trying to connect to Docker daemon`

**Solution (Linux):**
```bash
sudo usermod -aG docker $USER

# Log out and back in, or:
newgrp docker

# Verify
docker ps
```

**macOS:** No action needed (Docker Desktop handles permissions)

</Accordion>

<Accordion title="Container Won't Start">

**Symptom:** Container exits immediately or won't start

**Debug:**

```bash
# Check execution logs
prlt execution list
prlt execution logs <execution-id>

# Check Docker logs
docker logs <container-id>

# View container details
prlt docker list
docker inspect <container-id>
```

**Common causes:**

1. **Missing environment variables:**
   ```bash
   # Check required vars
   echo $GITHUB_TOKEN
   echo $GH_TOKEN
   
   # Set if missing
   export GH_TOKEN=$(gh auth token)
   export GITHUB_TOKEN=$(gh auth token)
   ```

2. **Volume mount path doesn't exist:**
   ```bash
   # Check agent directory exists
   ls -la agents/staff/alice/
   ```

3. **Port already in use:**
   ```bash
   # Find process using port
   lsof -i :PORT
   ```

4. **Insufficient resources:**
   ```bash
   # Check Docker resources
   docker stats
   
   # Increase in Docker Desktop:
   # Settings → Resources → Memory/CPU
   ```

</Accordion>

<Accordion title="Out of Disk Space">

**Symptom:** `no space left on device`

**Solution:**

```bash
# Check Docker disk usage
docker system df

# Clean unused resources
prlt docker prune
docker system prune -a

# Remove specific containers
prlt docker clean

# Remove unused volumes
docker volume prune

# Remove build cache
docker builder prune -a
```

**Increase Docker disk allocation (macOS):**
1. Docker Desktop → Settings → Resources
2. Increase Virtual Disk Limit
3. Apply & Restart

</Accordion>

<Accordion title="Volume Mount Failures">

**Symptom:** Files not visible in container

**Check:**

```bash
# Verify source path exists
ls -la /path/to/repo

# Check if path is in allowed Docker paths
# Docker Desktop → Settings → Resources → File Sharing
```

**macOS specific:**
1. Open Docker Desktop
2. Settings → Resources → File Sharing
3. Add path to allowed list
4. Apply & Restart

**Test mount:**
```bash
# Run test container with same mount
docker run --rm -v "$(pwd):/workspace" alpine ls -la /workspace
```

</Accordion>

## Git and Branch Issues

<Accordion title="Branch Pushed but No PR">

**Symptom:** Branch appears on GitHub but no pull request was created

**Causes:**

1. `--no-pr` was active
2. No `--create-pr` flag and no workspace default
3. `gh` CLI not installed or not authenticated

**Quick Fix:**

```bash
# Create PR after the fact
prlt pr create TKT-001
```

**Prevention:**

```bash
# Set workspace default
prlt config set execution.create_pr_default true

# Or use flag explicitly
prlt work start TKT-001 --create-pr
```

**PR Mode Resolution Order:**

1. `--create-pr` flag → create PR
2. `--no-pr` flag → skip PR
3. Non-code-modifying action → skip PR
4. Workspace config → use configured default
5. Interactive prompt

</Accordion>

<Accordion title="Worktree Already Exists">

**Symptom:** `fatal: 'agents/staff/alice/repo' already exists`

**Solutions:**

```bash
# Remove stale worktree
git worktree remove agents/staff/alice/repo --force

# Or rebuild agent
prlt agent rebuild alice

# List all worktrees
git worktree list
```

**Automatic cleanup:**
```bash
# Prune deleted worktrees
git worktree prune
```

</Accordion>

<Accordion title="Branch Already Exists">

**Symptom:** `fatal: A branch named 'feat/alice/TKT-001-feature' already exists`

**Solutions:**

1. **Force start (overwrites branch):**
   ```bash
   prlt work start TKT-001 --force
   ```

2. **Delete old branch:**
   ```bash
   git branch -D feat/alice/TKT-001-feature
   ```

3. **Delete remote branch:**
   ```bash
   git push origin --delete feat/alice/TKT-001-feature
   ```

</Accordion>

<Accordion title="Detached HEAD State">

**Symptom:** Agent can't commit, "HEAD detached"

**Solution:**

```bash
# In agent worktree
cd agents/staff/alice/repo

# Create branch from current state
git checkout -b feat/alice/TKT-001-feature

# Or checkout existing branch
git checkout feat/alice/TKT-001-feature
```

</Accordion>

<Accordion title="Merge Conflicts">

**Symptom:** PR has merge conflicts

**Solution:**

```bash
# Checkout agent's branch
git checkout feat/alice/TKT-001-feature

# Update main
git fetch origin main

# Merge main
git merge origin/main

# Resolve conflicts in editor
# Then:
git add .
git commit -m "Resolve merge conflicts"
git push
```

**Using rebase:**
```bash
git fetch origin main
git rebase origin/main
# Resolve conflicts
git rebase --continue
git push --force-with-lease
```

</Accordion>

## Agent Issues

<Accordion title="Agent Not Found">

**Symptom:** `Agent 'alice' not found`

**Solutions:**

```bash
# List all agents
prlt agent list
prlt agent staff list
prlt agent temp list

# Add missing agent
prlt agent staff add alice

# Or create ephemeral agent
prlt work start TKT-001 --ephemeral
```

</Accordion>

<Accordion title="Agent Stuck in Progress">

**Symptom:** Agent shows as working but nothing happening

**Solutions:**

1. **Check execution status:**
   ```bash
   prlt execution list
   prlt execution logs <execution-id>
   ```

2. **Stop and restart:**
   ```bash
   prlt execution stop <execution-id>
   prlt work start TKT-001 --force
   ```

3. **Kill container:**
   ```bash
   prlt docker stop alice
   # Or
   docker stop <container-id>
   ```

4. **Kill tmux session:**
   ```bash
   tmux kill-session -t TKT-001-implement-alice
   ```

</Accordion>

<Accordion title="Ephemeral Agent Creation Fails">

**Symptom:** `EACCES: permission denied, mkdir 'agents/temp'`

**Solution:**

```bash
# Create directory manually
mkdir -p agents/temp
chmod 755 agents/temp

# Verify
ls -la agents/
```

</Accordion>

## Ticket Issues

<Accordion title="Ticket Not Found">

**Symptom:** `Ticket TKT-001 not found`

**Solutions:**

```bash
# List all tickets
prlt ticket list

# Include all statuses
prlt ticket list --status all

# Search by project
prlt ticket list --project PROJ-001

# Search by text
prlt ticket list | grep "keyword"
```

</Accordion>

<Accordion title="Can't Move Ticket">

**Symptom:** Status move fails or invalid status

**Solutions:**

```bash
# Check valid statuses
prlt status list

# Move to valid status
prlt ticket move TKT-001 "In Progress"

# View ticket current status
prlt ticket view TKT-001
```

</Accordion>

## Network and API Issues

<Accordion title="GitHub Authentication Failed">

**Symptom:** `gh: authentication failed`

**Solutions:**

```bash
# Re-authenticate
gh auth logout
gh auth login

# Verify status
gh auth status

# Refresh token
gh auth refresh

# For prlt
prlt gh login
prlt gh status

# Export token
export GITHUB_TOKEN=$(gh auth token)
```

</Accordion>

<Accordion title="API Rate Limited">

**Symptom:** `API rate limit exceeded`

**Solutions:**

```bash
# Check rate limit status
gh api rate_limit

# Ensure authenticated (higher limits)
gh auth status
export GITHUB_TOKEN=$(gh auth token)

# Wait for rate limit reset
# Or use different authentication
```

**GitHub rate limits:**
- Authenticated: 5,000 requests/hour
- Unauthenticated: 60 requests/hour

</Accordion>

<Accordion title="Claude API Errors">

**Symptom:** Agent fails with API errors

**Solutions:**

```bash
# Check API key
echo $ANTHROPIC_API_KEY

# Test API access
curl https://api.anthropic.com/v1/messages \
  -H "x-api-key: $ANTHROPIC_API_KEY" \
  -H "content-type: application/json" \
  -d '{"model":"claude-3-sonnet-20240229","max_tokens":10,"messages":[{"role":"user","content":"Hi"}]}'

# Check Claude Code auth
claude --version

# Re-authenticate
claude login
```

**In containers:**
```bash
# Check credentials volume
docker volume inspect claude-credentials

# Verify credentials copied
prlt agent login alice
```

</Accordion>

## Performance Issues

<Accordion title="Slow CLI Response">

**Causes and solutions:**

1. **Large database:**
   ```bash
   # Check database size
   ls -lh .proletariat/workspace.db
   
   # Archive old tickets
   prlt ticket list --status Done | wc -l
   prlt ticket move TKT-old Canceled
   ```

2. **Many containers:**
   ```bash
   # Check container count
   docker ps -a | wc -l
   
   # Clean up
   prlt docker prune
   ```

3. **Disk I/O:**
   ```bash
   # Check disk usage
   df -h
   
   # Check inode usage
   df -i
   ```

</Accordion>

<Accordion title="High Memory Usage">

**Monitor:**
```bash
# Docker stats
docker stats

# System memory
top
htop
```

**Solutions:**

1. **Limit concurrent agents:**
   ```bash
   # Stop unused executions
   prlt execution list
   prlt execution stop <execution-id>
   ```

2. **Reduce container memory:**
   ```bash
   # Edit .devcontainer/devcontainer.json
   "runArgs": ["--memory=2g"]
   ```

3. **Increase Docker allocation:**
   - Docker Desktop → Settings → Resources
   - Increase Memory
   - Apply & Restart

</Accordion>

## Quick Fixes Checklist

When something isn't working, check:

- [ ] Is Docker running? `docker info`
- [ ] Am I in the right directory? `pwd && ls .proletariat/`
- [ ] Is PRLT_HQ_PATH set correctly? `echo $PRLT_HQ_PATH`
- [ ] Is the agent/ticket valid? `prlt agent list && prlt ticket list`
- [ ] Are there stale containers? `prlt docker clean`
- [ ] Is the database intact? `prlt workspace list`
- [ ] Is gh authenticated? `gh auth status`
- [ ] Are credentials set? `echo $GITHUB_TOKEN`
- [ ] Is Claude authenticated? `claude --version`
